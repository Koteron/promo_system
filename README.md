# README

## Общее описание

Данный проект представляет собой backend-сервис на Django 4.2+, реализующий создание заказов с возможностью применения промокодов.

Система включает основные сущности: пользователи, товары, заказы и промокоды. Ключевая функциональность — создание заказа через HTTP API с корректным расчётом стоимости и применением бизнес-правил промокодов.


## Технологии

- Python 3.10+
- Django 4.2+
- PostgreSQL
- Docker / Docker Compose
- uv (управление зависимостями и окружением)

## Установка и запуск

### 1. Установить uv

```
pip install uv
```

### 2. Установка зависимостей
```
uv sync --frozen --all-extras
```
После выполнения будет создано виртуальное окружение и установлены зависимости.

Активируйте окружение.

---

### 3. Запуск базы данных
```
docker compose up -d postgres
```
Запускает контейнер PostgreSQL.

---

### 4. Применение миграций
```
python manage.py migrate
```
Создаёт необходимые таблицы в базе данных.

---

### 5. Заполнение тестовыми данными
```
python manage.py seed_data
```
Добавляет начальные данные:
- пользователей  
- товары  
- промокоды  

<i> В том числе здесь создаётся `superuser` с логином и паролем: `admin`</i>

---

### 6. Сбор статических файлов (опционально)

Требуется при запуске с DEBUG=False, чтобы не ломались стили у админской панели:
```
python manage.py collectstatic
```
---

### 7. Запуск приложения
```
docker compose up -d web
```
После запуска сервис доступен по адресу:

`http://localhost:8000`

## API

### Создание заказа
```
POST /api/orders/create/

Пример запроса:

{
  "user_id": 1,
  "goods": [
    { "good_id": 1, "quantity": 2 }
  ],
  "promo_code": "SUMMER2025"
}

Пример ответа:

{
  "user_id": 1,
  "order_id": 1,
  "goods": [
    {
      "good_id": 1,
      "quantity": 2,
      "price": 100,
      "discount": "0.1",
      "total": 180
    }
  ],
  "price": 200,
  "discount": "0.1",
  "total": 180
}
```


## Правила применения промокодов

Промокод применяется только при выполнении всех условий:

- Промокод существует в системе  
- Промокод активен  
- Промокод не просрочен  
- Не превышен лимит использования  
- Один пользователь может использовать промокод только один раз  
- Промокод может быть ограничен категорией товаров  
- Некоторые товары могут быть исключены из акций  

Скидка рассчитывается как процент и применяется только к товарам, участвующим в акции.

## Django Admin

Административная панель доступна по адресу:

/admin/

Через неё можно:

- управлять пользователями  
- редактировать товары  
- создавать и изменять промокоды  
- просматривать заказы  
- отслеживать использование промокодов  
- анализировать состояние системы  

Админка используется для наблюдения за состоянием системы и проверки данных.

## Архитектура и особенности

- Бизнес-логика вынесена из views в сервисный слой  
- Валидация выполняется на уровне сериализаторов и сервисов  
- Все операции создания заказа выполняются атомарно  
- Учёт использования промокодов защищает от повторного применения  
- Логика скидок централизована и расширяема

- консистентность данных
